## SURFACE-BY-SURFACE: THE EXACT PATHWAY

### 1. TWITTER/X — THE MOST MEASURABLE SURFACE

**What We Can Get Publicly (No API):**

T1 (100% accessible):
  - Profile name, bio, location, URL
  - Follower count, following count, tweet count
  - Profile image, banner image
  - Join date

T2 (95% accessible via nitter/alternative frontends or scrape):
  - Last ~3200 tweets (Twitter's hard limit for public profiles)
  - Tweet text, timestamp, reply count, retweet count, like count
  - Media attachments (images, videos, cards)
  - Hashtags, mentions, URLs

T3 (90% accessible):
  - Quote tweet text (the comment + the embedded tweet)
  - Reply thread structure (who replied to whom)
  - Retweet vs. quote tweet distinction

T4 (NOT accessible without API):
  - Bookmarks (private action)
  - "See more" engagement (algorithmic, not public)
  - Impressions (only visible to tweet owner)
  - Profile visits driven by tweet

**The 90-Day Window Method:**

We fetch tweets in batches, stopping at the 90-day boundary. For each tweet, we extract:
- id, text, created_at, likes, replies, retweets, quote_count
- is_reply, is_retweet, is_quote
- media_urls, mentions, hashtags, urls

**Categorization Pipeline (The Shot-Charting):**

Each tweet is categorized into the NBA shot framework:
- 3-POINTER (Promo): Contains 'subscribe', 'try', 'join the', 'waitlist', 'download', product names, or explicit CTAs
- ASSIST: Is a retweet or mentions another Every team member
- MID-RANGE (Philosophy): Contains thesis words like 'the future of', 'framework', 'pattern', 'insight'
- FREE THROW (Personal): Contains no work signals (AI, product, building, startup, code)
- AIRBALL: Forced promo with engagement below 50% of player's median

**Engagement Metric Calculation:**

- FG% = Total Points / (Total Posts * Follower Count) * 1000
- 3P% = Promo Likes / Promo Posts / Follower Count * 1000
- FT% = Personal Likes / Personal Posts / Follower Count * 1000
- TS% = Total Points / (Total Posts * max(Follower Count, 1)) * 100
- Assist% = Assist Posts / Total Posts

**The Bookmark Problem:**
- Bookmarks are private. We cannot see who bookmarked a tweet.
- Workaround: Use bookmarks as a proxy for "high-intent engagement." If a tweet has high likes + high replies + low retweets, it likely has high bookmarks (people saving for later rather than sharing).
- Better Workaround: Ask the player to screenshot their Twitter Analytics dashboard (impressions, profile visits, link clicks). This is internal data that makes Surface IQ 10x more accurate.

### 2. LINKEDIN — THE ENTERPRISE BLACK BOX

**What We Can Get Publicly:**

T1 (100% accessible):
  - Name, headline, current role, past roles
  - Connection count (if public)
  - Profile photo, banner, featured section
  - About section text

T2 (60% accessible — heavily rate-limited):
  - Public posts (last ~50-100)
  - Post text, timestamp, reaction count (likes only)
  - Comment count (but not comment text)
  - Share count

T3 (NOT accessible):
  - Who viewed the profile
  - Private messages generated by post
  - InMail conversion rates
  - Detailed analytics (impressions, click-throughs)

**The LinkedIn Scraping Challenge:**
LinkedIn aggressively blocks scraping. The reliable methods:
1. LinkedIn API (requires OAuth, company partnership, or Sales Navigator)
2. Manual collection (open profile, scroll, record)
3. Proxy rotation (technically against ToS)
4. Ask the employee for a data export (they can download their own activity)

**Surface IQ Workaround for LinkedIn:**
Since we can't get granular engagement, we measure presence quality instead of engagement quantity:
- Does the profile mention Every in the headline? (binary)
- Is the featured section active? (binary)
- How many posts in the last 90 days? (count)
- Are posts about AI/enterprise (relevant) or personal (off-topic)? (categorical)
- Does the profile link to Every products? (binary)

### 3. SUBSTACK — THE PARTIALLY OPEN NEWSLETTER

**What We Can Get Publicly:**

T1 (100% accessible):
  - Publication name, subscriber count (if displayed)
  - Post titles, publication dates
  - Free post content (full text)
  - Author bio, about page

T2 (60% accessible):
  - Like count on posts (visible at bottom of post)
  - Comment count (visible)
  - Public comment text (if comments are open)
  - Restack count (Substack's retweet equivalent)

T3 (NOT accessible):
  - Subscriber-only post content (paywalled)
  - Subscriber-only comments
  - Email open rates, click-through rates
  - Paid subscriber conversion data
  - "Like" details (who liked it)

**The Substack Comment Like Problem:**
- Substack shows total likes on a post.
- Substack shows public comments and likes on those comments (if you're logged in and the commenter is public).
- We cannot see subscriber-only comments or their likes.
- Workaround: For Austin's Substack (@tedescau, 34K+ subscribers), we can see public Notes (Substack's Twitter-like feature) and their engagement. We can see free post engagement. But the granular "who liked what comment" is invisible.

### 4. GITHUB — THE MOST TRANSPARENT SURFACE

**What We Can Get Publicly:**

  - Profile name, bio, location, company
  - Public repo count, follower count, following count
  - Contribution graph (last 365 days)
  - Repo names, descriptions, star counts, fork counts

T2 (100% accessible via API):

**GitHub is the easiest surface to measure** because the API is generous and doesn't require auth for public data.

## THE COMPLETE DATA PIPELINE

Phase 1: DISCOVERY
  Input: Company name, team page
  Tools: web_search, every.to/team scrape, byline mining
  Output: Roster of names + roles + suspected handles

Phase 2: HANDLE VERIFICATION
  Input: Suspected handles
  Tools: Direct URL check, bio text matching, cross-reference
  Output: Verified handle map per surface per employee

Phase 3: DATA HARVESTING
  Input: Verified handles
  Tools:
    - Twitter: nitter instances, twitter-scraper, twscrape
    - LinkedIn: manual collection (scraping blocked)
    - GitHub: github.com API (no auth needed)
    - Substack: page scrape (public posts only)
    - Product Hunt: page scrape
    - YouTube: YouTube Data API or page scrape
  Output: Raw engagement data (likes, replies, retweets, etc.)

Phase 4: CONTENT CATEGORIZATION
  Input: Raw post text + metadata
  Tools: Keyword matching, LLM classification, manual review
  Output: Shot type (3P, mid, FT, assist, airball) per post

Phase 5: METRIC CALCULATION
  Input: Categorized posts + engagement data
  Tools: Pandas, custom formulas (FG%, 3P%, FT%, TS%, Assist%)
  Output: Player card metrics

Phase 6: INSIGHT GENERATION
  Input: Metrics + surface coverage map
  Tools: Comparative analysis, gap detection, opportunity scoring
  Output: Surface IQ report + recommendations

## THE HONEST LIMITATIONS

### What We CANNOT Do Without API Keys / Internal Access:

1. Twitter Bookmarks — Private. Invisible.
2. Twitter Impressions — Only visible to tweet owner via Twitter Analytics.
3. Twitter "See More" / Algorithmic Reach — Not public.
4. LinkedIn Detailed Engagement — Heavily rate-limited, requires Sales Navigator.
5. Substack Subscriber-Only Comments — Paywalled.
6. Substack Email Open Rates — Internal only.
7. GitHub Private Repo Activity — Invisible.
8. GitHub Traffic Data — Requires repo admin access.
9. Product Hunt Conversion Data — Internal only.
10. YouTube Watch Time / Retention — Requires channel ownership.

### What We CAN Do (And What Makes Surface IQ Work Anyway):

1. Relative Performance — We don't need absolute impressions. We need "how did this tweet perform vs. this player's median?"
2. Content Mix Analysis — We can categorize every public post without needing private data.
3. Cross-Surface Correlation — We can see that Kieran's philosophy tweets outperform his promo tweets using only public like counts.
4. Absence Detection — We can find missing surfaces (Product Hunt makers, absent newsletters) with zero private data.
5. Trend Detection — We can spot "this player is posting more 3s lately" using only public timelines.

## SUMMARY: THE FINITE ANSWER

Q: Can we get "all the way down to likes on a Substack comment"?

A:
- Substack post likes: YES (visible on public posts)
- Substack comment likes: PARTIALLY (only on public comments, and only if logged in)
- Substack subscriber-only comment likes: NO (invisible to non-subscribers)
- Twitter bookmarks: NO (private)
- Twitter quote tweet text: YES (visible)
- Twitter quote tweet engagement: YES (likes/replies visible)
- LinkedIn detailed analytics: NO (requires internal access)
- GitHub everything public: YES (API is open)

The Surface IQ thesis doesn't require infinite granularity. It requires:
1. Presence detection (are you on this surface?)
2. Content categorization (what are you posting?)
3. Relative engagement (how does this post perform vs. your baseline?)
4. Cross-surface attribution (where are you missing?)

All four work with 100% public data.
